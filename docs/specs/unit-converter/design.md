# Design

## Architecture

- `lib/pages/radio/unit_converter_page.dart`
  - 使用局部 `StatefulWidget` 管理两个模式的输入与联动。
  - 采用顶部模式切换 + 参数区 + 双列输入区的工具型布局。
- `lib/services/unit_converter_service.dart`
  - 负责单位定义、格式化和换算公式。
  - 暴露两类结果：
    - `PowerVoltageConversionResult`
    - `FieldFluxConversionResult`

## Data Flow

1. 页面初始化默认进入“电压功率换算”，使用 `13.8V`、`50Ω` 生成首屏值。
2. 用户切换到“场强通量密度换算”时，页面根据 `1V/m`、`50MHz`、`50Ω` 生成首屏值。
3. 用户编辑任意输入框后，页面解析当前值并调用 service。
4. service 返回该模式下全部字段结果，页面同步更新非当前编辑字段。
5. 输入为空、非数字或小于等于零时，页面展示错误提示。

## Conversion Models

### 电压功率换算

- 电压与功率关系：
  - `P = V^2 / R`
  - `V = sqrt(P * R)`
- 电压分贝：
  - `dBV = 20log10(V)`
  - `dBmV = 20log10(V / 1mV)`
  - `dBμV = 20log10(V / 1μV)`
- 功率分贝：
  - `dBW = 10log10(P)`
  - `dBmW = 10log10(P / 1mW)`
  - `dBμW = 10log10(P / 1μW)`

### 场强通量密度换算

- 自由空间平面波关系：
  - `S = E^2 / Z0`
  - `Z0 = 120π`
- 频率换算波长：
  - `λ = c / f`
- 接收功率使用等效口径近似：
  - `Pr = S * Ae`
  - `Ae = λ^2 / (4π) * (R / 50)`

说明：

- 这里的功率是基于输入频率和阻抗推导出的等效接收功率，属于工程估算，不代表具体天线系统的完整链路结果。
- 为了贴近用户给出的参考界面，场强模式保留频率与阻抗两个显式参数。

## UI

- 顶部使用两个大按钮切换模式：
  - `电压功率换算`
  - `场强通量密度换算`
- 参数区：
  - 电压模式仅显示 `阻抗`
  - 场强模式显示 `频率` 与 `阻抗`
- 内容区按分组显示：
  - 电压模式：`电压`、`功率`
  - 场强模式：`场强`、`通量密度`、`功率`
- 每组使用双列输入网格，小屏自动退化为单列。
- 单位标签用 TeX 渲染，确保 `μ`、上标 `²` 等显示一致。

## API Contract

无接口改动。

## Storage & Permissions

- 不新增 Secure Storage 或 SQLite 数据。
- 不涉及定位、传感器或文件权限。

## Radio Compliance

- 页面底部固定显示“仅供参考，请遵守当地法规和主管部门要求。”
- 所有结果仅作为工程估算辅助，不表述为法规、认证或官方结论。

## Testing

- service 测试覆盖：
  - 电压 / 功率双向换算
  - 场强 / 通量密度换算
  - 场强 / 功率在指定频率和阻抗下的联动
- 页面测试覆盖：
  - 双模式核心区块渲染
  - 切换到场强模式后显示频率和阻抗参数
  - 编辑输入框时当前光标位置保持稳定
