# QSO Logbook Architecture Requirements

## 背景

当前项目已经具备本地 QSO 记录、云端同步、QSL 公开确认和若干无线电工具能力，但日志相关能力仍以单表和单页面为主，后续若继续增加 ADIF 导入导出、台站资料、批量录入、LoTW 查询与上传等能力，现有结构会很快变得难以维护。

本需求旨在先梳理日志子系统的产品边界和技术边界，为后续迭代建立稳定的数据模型、页面结构和服务分层，避免在现有 `RadioLogPage` 与 `QsoManagementService` 上继续堆叠职责。

## 用户故事

- 作为用户，我希望把通联记录按日志本或场景组织，而不是所有记录都混在一个列表里，便于管理不同年份、活动或设备的日志。
- 作为用户，我希望主页或主 tab 里的日志区域保持轻量，只做最近日志预览和快速写入，避免把完整日志管理塞进主导航。
- 作为用户，我希望为不同操作位置维护独立的台站资料，这样录入、导出和后续上传时不需要反复填写相同信息。
- 作为用户，我希望能导入和导出 ADIF 文件，方便与其他业余无线电日志工具互通。
- 作为用户，我希望有一套更高效的批量录入体验，能快速把常见通联文本转换为结构化 QSO。
- 作为用户，我希望在不泄露敏感凭据的前提下查询 LoTW 状态，并在后续支持受控的上传流程。
- 作为用户，我希望完整的 LoTW 日志管理在“工具”页有独立入口，这样我能在需要时进入完整功能，而不是干扰日常主流程。
- 作为维护者，我希望日志能力的本地存储、服务编排、页面入口和后端边界清晰可控，避免未来改动互相牵连。

## 范围

### 包含

- 为无线电日志子系统建立独立 spec，明确页面、模型、服务和存储边界。
- 将现有“单一 QSO 列表”演进为“日志本 + QSO 明细”的信息架构。
- 调整日志相关页面分层：主页面 tab 仅保留快速预览与快速写入，完整日志/LoTW 管理作为工具页独立页面。
- 引入独立的台站资料模型，用于承载本台呼号、网格、DXCC、行政区、备注等字段。
- 规划 ADIF 导入、导出、批量解析和后续上传所需的数据字段与服务职责。
- 规划 LoTW 查询、状态展示和后续上传流程的页面入口、凭据存储和错误处理。
- 规划本地 SQLite 迁移与 Secure Storage 使用边界。
- 明确与 `beacon-api`、MeowzExam API、OAuth 的关系和不允许跨越的边界。

### 不包含

- 本次 spec 不直接实现完整 LoTW 证书签名能力。
- 本次 spec 不修改 `../beacon-api`、`../MeowzExam`、`../mp-oauth2`。
- 本次 spec 不承诺所有平台在第一阶段具备完全一致的日志高级能力。
- 本次 spec 不引入公共 QSO 分享、他人日志浏览或日志公开检索。

## 验收标准

- 存在一套独立 spec，覆盖 requirements、design、tasks 三份文档。
- spec 明确“日志本”“台站资料”“QSO 记录”“外部状态查询/上传”四类对象的职责边界。
- spec 明确主页面 tab、工具页入口、完整日志管理页之间的职责划分。
- spec 明确哪些数据进入 SQLite，哪些数据进入 Secure Storage，哪些数据不得进入后端。
- spec 明确证书、私钥、`.p12` 导入文件、签名流程和备份流程的安全边界。
- spec 明确 ADIF、快速录入、LoTW 查询与上传流程分别由哪些服务承载。
- spec 明确现有 `RadioLogPage` 后续如何拆分或重组为更清晰的页面结构。
- spec 明确第一阶段与后续阶段的实施顺序，避免一次性大改。

## Impact

- MeowzExam API: 无，题库、练习、考试和 AI 解析接口不受本需求影响。
- beacon-api: 有，后续可能新增非敏感的日志辅助接口或状态同步接口，但本 spec 不要求立即修改接口契约。
- OAuth: 无，继续只负责登录授权，不扩展为日志签名或第三方凭据托管。
- Local storage / SQLite: 有，需要新增日志本、台站资料、导入元数据、外部状态追踪等表或字段，并设计迁移策略。
- Platform config: 有，文件导入导出和后续平台差异能力需要在 Android、桌面、Web 上分别评估。

## Open Questions

- 第一阶段是否需要“日志本”这一层真实落库，还是先用默认日志本兼容旧数据。
- LoTW 查询是否在第一阶段就面向所有平台开放，还是先限移动端或桌面端。
- 现有 QSO 云同步是否继续以“单记录直传”方式工作，还是要引入日志本上下文。
- ADIF 导入后的重复记录判定规则应以哪些字段作为主键指纹。
- 平台安全存储能力不足的平台是否直接禁用本地签名，仅保留导入导出与查询。
