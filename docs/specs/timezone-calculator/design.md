# Design

## Architecture

- 新增 `TimezoneCalculatorPage` 作为独立工具页面，放在 `lib/pages/radio/`。
- 新增轻量模型与离线 helper，负责地点、UTC 偏移、昼夜判断和时差结果计算。
- 页面状态使用局部 `StatefulWidget` 管理，不引入新的全局 Provider。
- 地图层复用现有 `flutter_map` 和天地图 / OSM 切换模式。

## Data Flow

1. 页面初始化时创建两个默认地点和当前 UTC 时间。
2. 用户通过地图点击或手动输入经纬度更新地点 A / B。
3. 离线 helper 根据经纬度推导近似 UTC 偏移。
4. 页面根据当前时间与 UTC 偏移计算两地本地时间、时差和日期进位。
5. 地图根据当前时间生成昼夜遮罩，并叠加近似时区带和地点 marker。
6. 用户切换底图、互换地点或使用定位后，页面即时刷新结果。

## API Contract

无接口改动。

## UI States

- Loading: 地图底图加载与初始布局显示。
- Empty: 未形成有效双地点时，结果区提示继续选择地点。
- Error: 经纬度格式错误、定位失败、天地图 token 缺失时显示明确提示。
- Permission denied: 定位权限未授权或定位服务未开启时显示提示。
- Success: 显示两地时间、时差、时区带和昼夜地图。

## Storage & Permissions

- 不新增 Secure Storage 或 SQLite 数据。
- 如用户使用“当前定位”，需请求定位权限并处理拒绝、永久拒绝和服务未开启状态。
- 未授权时不阻塞手动输入和地图点击流程。

## Radio Compliance

- 页面底部固定显示“仅供参考，请遵守当地法规和主管部门要求。”
- 时区、白天 / 黑夜和本地时间展示仅为近似工具结果，不表述为官方结论。

## Testing

- 自动化测试覆盖 UTC 偏移推导、半小时 / 45 分钟偏移映射、时差换算、日期进位、昼夜判断基础样例。
- 手动验证工具入口、地图点击选点、双地点换算、宽屏 / 小屏布局、定位权限拒绝和底图切换。
